Spring IoC and DI Demo

Learn how Spring wires objects together through the IoC container without forcing your classes to create their own dependencies.

The DAO bean comes from XML, while the service and controller come from annotations.

Goal: understand how a bean defined in applicationContext.xml can be injected into a class discovered by @Component.

How It Works

Spring wiring in four simple steps

01

XML registers the DAO bean

applicationContext.xml tells Spring to create userDao.

02

The container manages the objects

Spring stores beans and decides how dependencies should be connected.

03

The service receives the DAO

UserService gets UserDao through constructor injection.

04

The browser shows the final result

The controller reads the service output and renders this page with Thymeleaf.

Study Guide

Key parts of the project

  • applicationContext.xml defines the XML-managed bean.
  • UserDaoImpl supplies sample user data.
  • UserService shows annotation-based dependency injection.
  • HomeController prepares the data for the browser.

Why This Matters

Loose coupling makes code easier to test and maintain

Instead of creating dependencies manually, each class focuses on its own responsibility and lets Spring handle the object lifecycle. That is the core idea behind IoC and dependency injection.
